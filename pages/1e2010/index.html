# 访问修饰符

java中有四种访问修饰符：private、default(无相应保留字)、protected和public。其含义如下：

  • private：如果一个元素声明为private，则元素的访问权限仅限于当前类。
  • default：如果一个元素声明为default，则元素的访问权限仅限于当前包。
  • protected：如果一个元素声明为protected，同一个包内的类可以访问，并且其他包内这个类的子类也可以访问。
    • 相当于比default多了一个其他包的子类可以访问
  • public：如果一个元素声明为public，则所有位置都可以调用元素（无论是在同一类中还是在同一包内）。

四种访问修饰符合元素的访问限制，从强到弱依次是private、default、protected、public。如果类A和类B的访问修饰符是public，当类A中的一个方法想要调用类B中的一个方法时，可以根据下图确定可访问性:

public class DefaultSingletonBeanRegistry implements SingletonBeanRegistry {
    private Map<String, Object> singletonObjects = new ConcurrentHashMap<>();
    @Override
    public Object getSingleton(String beanName) {
        return singletonObjects.get(beanName);
    }
    protected void addSingleton(String beanName, Object singletonObject) {
        singletonObjects.put(beanName, singletonObject);
    }
}

# 抽象类定义模板方法(AbstractBeanFactory)

public abstract  class AbstractBeanFactory extends DefaultSingletonBeanRegistry implements BeanFactory {
    /**
     * 有bean则返回，没有的话根据其beanDefinition创建一个
     */
    @Override
    public Object getBean(String name) throws BeansException{
        //先去单例对象集合查询是否有，如果没有，会在单例对象集合创建一个Bean
        Object bean = getSingleton(name);
        if(bean != null){
            return bean;
        }
        BeanDefinition beanDefinition = getBeanDefinition(name);
        return createBean(name,beanDefinition);

    }
    protected abstract BeanDefinition getBeanDefinition(String beanName) throws BeansException;

    protected abstract Object createBean(String beanName,BeanDefinition beanDefinition) throws BeansException;

}

# 实例化Bean类(AbstractAutowireCapableBeanFactory)

public abstract class AbstractAutowireCapableBeanFactory extends AbstractBeanFactory {
    @Override
    protected Object createBean(String beanName, BeanDefinition beanDefinition) throws BeansException {
        Object bean = null;
        try {
            bean = beanDefinition.getBeanClass().newInstance();
        } catch (InstantiationException | IllegalAccessException e) {
            throw new BeansException("Instantiation of bean failed", e);
        }
        addSingleton(beanName, bean);
        return bean;
    }
}

DefaultListableBeanFactory

public class DefaultListableBeanFactory extends AbstractAutowireCapableBeanFactory implements BeanDefinitionRegistry{
    private final Map<String, BeanDefinition> beanDefinitionMap = new ConcurrentHashMap<>();

    @Override
    public void registerBeanDefinition(String beanName,BeanDefinition beanDefinition){
        beanDefinitionMap.put(beanName,beanDefinition);
    }

    @Override
    public BeanDefinition getBeanDefinition(String beanName) throws BeansException{
        BeanDefinition beanDefinition = beanDefinitionMap.get(beanName);
        if(beanDefinition==null) throw new BeansException("No bean named "+beanName+" is defined");
        return beanDefinition;
    }
}

# STAR法则总结

2024-01-24，第一次复习总结

# situation

上节的springBean容器只是一个引子，非常不完善，也很混杂，现在将其中的Bean定义，获取等完善，并且将这些功能分离独立出来，建立Bean体系结构。

# task

1.采用接口-抽象类-实现类的体系完成功能之间的分离

2.引入单例对象体系

# action

BeanFactory体系

  1. 将BeanFactory变成接口，只提供getBean()即获取对象功能。
  2. AbstractBeanFactory抽象Bean工厂类，定义BeanFactory需要实现功能的逻辑（并不一定直接实现），具体的函数会继续交给下一级的类实现
  3. AbstractAutowireCapableBeanFactory抽象Bean自动装配类，提供创建Bean的流程，（并不完善，这个类后面还有很多内容）
    1. 这里还会在生成Bean时，调用addSingleton() 默认将生成的Bean对象添加到单例集合 singletonObjects
    2. 这里插一句，学到后面我们知道，Spring中最基本的Bean Scope就只有两种，单例和原型，并且大部分都是单例（通过scope属性可以配置自己定义的Bean）
  4. 将注册Bean定义独立出来，创建BeanDefinitionRegistry接口，提供registerBeanDefinition即注册BeanDefinition
  5. DefaultListableBeanFactoryBean工厂的实现类，继承自AbstractAutowireCapableBeanFactory，BeanDefinitionRegistry
    1. 拥有beanDefinitionMap，BeanDefinition缓存
    2. 提供registerBeanDefinition注册Bean定义的实现
    3. 提供getBeanDefinition,获取Bean定义

SingletonBeanRegistry体系

  1. 新增SingletonBeanRegistry单例对象接口，提供给getSingleton添加单例对象
  2. 新增DefaultSingletonBeanRegistry实现SingletonBeanRegistry单例Bean仓库接口
    1. 拥有Map<String,Object> singletonObjects,即单例对象缓存
    2. 提供getSingleton获得单例对象功能，addSingleton添加单例对象功能
