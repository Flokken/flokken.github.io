import java.util.*;

        Scanner sc = new Scanner(System.in);

        for(int i=0;i<n;i++){

        for(int i=0;i<s.length-1;i++){

        }

# 类型转换

当java在不同基本类型之间进行运算时

  • 如果是小类型与大类型进行计算，小类型会自动转换为大类型，这是隐性转换
    • 比如int->long,int->double
  • 如果是大类型转换为小类型，不能自动转换，需要手动强制转换，这是显式转换
    • long->int,dobule->int

例子

class Solution {
    public int countHousePlacements(int n) {
		int x=(int)1e9+7;//1e9默认是long类型
        int y=(int)1e9+8;
        //x，y都是int型，然后强转x为long，那么后面的y会自动提升为long
        //由于返回int，再将long强转为int
        return (int)((long)x*y);

    }
}

# 自动拆箱和自动装箱

首先我们复习一下包装类和基本数据类型

Question：为什么java要有包装类呢？

Answer：Java 是号称面向对象的语言，所有的类型都是引用类型。 Object 类是所有类的父类，而且是唯一不用指定明确继承的类。 但是基本类型如int 不是引用类型，也不是继承自Object，所以Java 需要一个这样的包装类来使其面向对象的完整性

装箱：把基本类型转换为其对应的包装类

  • 包装类是final类，一旦创建不可更改

拆箱：把包装类转变为其基本类型

在Java SE5之前，如果要生成一个数值为10的Integer对象，必须这样进行：

Integer i = new Integer(10);
1

如果要把Integer对象转变为int，需要

int j = Integer.valueOf(i);
1

这样做比较繁琐，所以jdk1.5之后引入了自动拆箱和自动装箱

一文带你理解Java中自动装箱和拆箱 (opens new window)

自动装/拆箱是指通过类型转换(隐式或显式)从值类型中获取引用类型，这部分工作是编译器帮我们来完成的。

比如：

# Lambda表达式

jdk8引入的新特性。

其实感觉lambda表达式就是更加简洁的匿名内部函数。

用处：当函数有一个函数型参数/接口时，传一个lambda函数进去就相当于传一个函数进去，也就是实现了这个接口

Lambda表达式通常用于函数式接口（Functional Interface），并且只有一个只有一个抽象方法的接口。Lambda表达式可以替代使用该接口的实现类，并且可以在需要函数作为参数的地方进行传递。Lambda表达式的语法形式为：

(parameters) -> expression或者(parameters) -> { statements; }

lambda有很多可以省略的地方，如下

  • Lambda 表达式的参数可以是零个(空参)或多个

  • 参数类型省略：参数的类型可以明确声明，也可以不声明，由 JVM 隐式的推断，例如 (int a) 和 (a) 效果相同

  • **省略括号：**当参数只有一个，且类型可推导时，可以省略括号，例如 (a) 与 a 效果相同

  • **省略函数体的花括号：**如果 Lambda 表达式的方法体只有一条语句时，可以省略花括号

  • **省略return：**如果 Lambda 表达式的方法体只有一条语句，且为返回值的时候，可以省略 return

# 方法引用

Lambda表达式也通常用于函数式接口（Functional Interface）。它允许直接引用已经存在的方法作为函数式接口的实现

  1. 静态方法引用：引用静态方法，例如 ClassName::staticMethodName。例如，Integer::parseInt 引用了 Integer 类的静态方法 parseInt。
  2. 实例方法引用：引用某个对象的实例方法，例如 instance::methodName。例如，str::length 引用了字符串对象 str 的 length 方法。
  3. 类的任意对象方法引用：引用某个类任意对象的实例方法，例如 ClassName::methodName。例如，String::toUpperCase 引用了 String 类的 toUpperCase 方法。
  4. 构造函数引用：引用构造函数来创建新对象，例如 ClassName::new。例如，ArrayList::new 引用了 ArrayList 类的构造函数。

很简单，就是类名::方法名，构造函数是类名::new

举例

List<Integer>转int[] 数组

int[] nums = nums1.stream().mapToInt(Integer::intValue).toArray();

# Stream流

Java 8引入了Stream流的概念，**它提供了一种流式操作集合的方式，**使得代码更简洁、可读性更高。

Stream是对集合数据进行操作的抽象。它可以通过一系列的中间操作和一个终端操作来实现对集合的处理。

创建Stream流：

  • 从Collection体系直接创建流（即List和Set）
  • Map体系集合，把Map转成Set集合，间接的生成流（不能直接生成）
  • 数组，通过Arrays中的静态方法stream生成流
  • 同种数据类型的多个数据，通过Stream接口的静态方法of(T... values)生成流

public class StreamDemo {
    public static void main(String[] args) {
        //Collection体系的集合可以使用默认方法stream()生成流
        List<String> list = new ArrayList<String>();
        Stream<String> listStream = list.stream();

        Set<String> set = new HashSet<String>();
        Stream<String> setStream = set.stream();

        //Map体系的集合间接的生成流
        Map<String,Integer> map = new HashMap<String, Integer>();
        Stream<String> keyStream = map.keySet().stream();
        Stream<Integer> valueStream = map.values().stream();
        Stream<Map.Entry<String, Integer>> entryStream = map.entrySet().stream();

        //数组可以通过Arrays中的静态方法stream生成流
        String[] strArray = {"hello","world","java"};
        Stream<String> strArrayStream = Arrays.stream(strArray);
              	//同种数据类型的多个数据可以通过Stream接口的静态方法of(T... values)生成流
        Stream<String> strArrayStream2 = Stream.of("hello", "world", "java");
        Stream<Integer> intStream = Stream.of(10, 20, 30);

    }
}

# Comparator比较器

Comparator是Java中的一个接口，位于java.util包中。它定义了一种比较两个对象的方式，使得可以根据自定义的排序规则对对象进行排序。Comparator接口包含一个compare方法，用于比较两个对象的顺序,通过实现Comparator接口并重写compare方法，可以定义自定义的比较逻辑。该方法具有以下定义：

interface Comparator<T> {
    int compare(T obj1, T obj2);
}
1
2
3

compare方法接受两个参数obj1和obj2，表示要比较的两个对象，返回一个整数值来表示它们的相对顺序。

  • 当返回负数时，表示obj1应该排在obj2之前；
  • 当返回正数时，表示obj1应该排在obj2之后；
  • 当返回零时，表示obj1和obj2相等。

Comparator是一个独立的接口，它需要通过创建一个独立的比较器实现类或使用匿名内部类(lambda表达式当然也行)来实现。这意味着你可以在不修改原始类的情况下定义多个不同的比较规则。

compareTo方法接受一个参数obj，表示要与当前对象进行比较的对象，返回一个整数值来表示它们的相对顺序。

  • 当返回负数时，表示当前对象应该排在obj之前；
  • 当返回正数时，表示当前对象应该排在obj之后；
  • 当返回零时，表示当前对象和obj相等。

通过实现Comparable接口并重写compareTo方法，可以定义对象的自然排序逻辑。

# 例子

场景：考虑一个表示学生的类Student，它具有name和age两个属性。我们可以使用Comparator和Comparable来定义对Student对象的比较规则。

实现Comparable接口，让Student对象根据学生年龄进行自然排序

public class Student implements Comparable<Student> {
    private String name;
    private int age;
    // 构造函数、getter和setter方法
    // 实现Comparable接口的compareTo方法来比较学生的年龄
    @Override
    public int compareTo(Student other) {
        return this.age - other.getAge();
    }
}

public class Main {
    public static void main(String[] args) {
        List<Student> students = new ArrayList<>();
        students.add(new Student("Alice", 20));
        students.add(new Student("Bob", 18));
        students.add(new Student("Charlie", 22));

        // 使用Comparable接口的自然排序
        Collections.sort(students);

        // 输出排序结果
        for (Student student : students) {
            System.out.println(student.getName() + " - " + student.getAge());
        }
    }
}

实现Comparator，即比较器接口，

这两者的区别就是，Comparator是更加独立的（而不是对象的），所以这里也是静态方法。

实际上，Comparator更常用的是在调用的方法需要传入一个比较器，此时用一个匿名内部类/lambda方法实现比较器。

import java.util.Comparator;

public class Student {
    private String name;
    private int age;
    // 构造函数、getter和setter方法
    // 定义一个Comparator实现类来比较学生的年龄
    public static class AgeComparator implements Comparator<Student> {
        @Override
        public int compare(Student s1, Student s2) {
            return s1.getAge() - s2.getAge();
        }
    }
}

public class Main {
    public static void main(String[] args) {
        List<Student> students = new ArrayList<>();
        students.add(new Student("Alice", 20));
        students.add(new Student("Bob", 18));
        students.add(new Student("Charlie", 22));

        // 创建一个基于年龄的比较器对象
        Comparator<Student> ageComparator = new Student.AgeComparator();

        // 使用比较器进行排序
        Collections.sort(students, ageComparator);

        // 输出排序结果
        for (Student student : students) {
            System.out.println(student.getName() + " - " + student.getAge());
        }
    }
}
