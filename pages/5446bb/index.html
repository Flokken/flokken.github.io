# 10进制转2进制

除2取余法，比如10， 10/2 =5， 0 5/2 = 2， 1（除不尽向下取整） 2/2= 1， 0 1/2=0, 1 ,所以结果是1010.

public int ten2two(int num){
    StringBuilder sb = new StringBuilder();
    while(num!=0){
        sb.append(num%2);//求余数
        num>>=1;//除以2
    }
    return sb.toString();
}

完整的例子：

import java.util.ArrayList;
import java.util.Collections;
import java.util.Comparator;
import java.util.List;

public class Main {
    public static void main(String[] args) {
        List<Long> sum = new ArrayList<>();
        sum.add(5L);
        sum.add(2L);
        sum.add(8L);
        sum.add(1L);

        // 降序排序
        Collections.sort(sum, (num1, num2) -> Long.compare(num2, num1));

        // 输出排序后的结果
        for (Long num : sum) {
            System.out.println(num);
        }
    }
}

import java.util.ArrayList;
import java.util.Collections;
import java.util.Comparator;

public class Main {
    public static void main(String[] args) {

                if (num1 > num2) {
                    return -1;

                    return 1;
                } else {
                    return 0;
                }
            }
        });

        // 输出排序后的结果
        for (Long num : sum) {
            System.out.println(num);
        }
    }
}

# Map按照值排序

如果想按照key来排序，用TreeMap就可以；

如果想实现按value排序，可以把map的键值对存到一个List<Map.Entry<String,Integer>>里面，然后对list进行排序。

public class Run{
    public static void main(String[] args) {
        Map<String,Integer> map = new HashMap<>();
        map.put("test1",1);
        map.put("test2",2);
        map.put("test3",3);
        map.put("test4",4);
        //降序排列
        List<Map.Entry<String,Integer>>ans = new ArrayList<>(map.entrySet());
        ans.sort((o1,o2)->(o2.getValue()-o1.getValue()));
        System.out.println(ans.get(0).getKey());
    }
}

二元组排序是经常遇到的需求，但是怎么快速实现呢？

例题：

1626. 无矛盾的最佳球队 (opens new window)

这里要对(age,socre)这个二元组排序，

  • 按照年龄从小到大排序
  • 如果年龄一样，按分数从小到大排序

最好想的想法是创建一个数组，有两个属性，然后对这个数组排序，得到这样一个数组。

但是写起来比较耗时，这里介绍灵神的写法。

我们观察，我们其实就是为了获得一个,也就是在排序后应该在哪个位置。

所以可以直接创建一个下标数组，然后排序时，实现lambda表达式时用age数组和score数组实现即可。

Integer []ids=new Integer[n];

Arrays.sort(ids,(a,b)->ages[a]!=ages[b]?ages[a]-ages[b]:scores[a]-scores[b]);

# 反转一个数组

假设有数组[1,2,3,4,5]，反转后就是[5,4,3,2,1]

怎么反转呢？假设有数组，其长度为，所以下标范围就是;

我们遍历时不断交换首尾元素直到即可（包括n/2）。

n是奇数或者偶数有影响吗？

分类讨论一下

  • 如果n是偶数，比如4，那么n/2=2, 交换1,4, 2,3。 正确

  • 如果n是奇数，比如5，那么n/2=2, 交换1,5, 2,4 。正确

    • 第三个元素不用交换，所以也可以

但是注意元素个数是1-n,但是下标是0-n-1，说明循环条件存在一个开闭区间的问题。

左闭右开：

下标范围, 循环条件是左闭右开区间写法

那么取值时应该是。注意这里由于n取不到，所以应该-1

for(int i=0;i<n/2;i++)  swap(a[i],a[n-i]);
1

左闭右闭：

下标范围, 循环条件是左闭右开区间写法

那么取值时应该是​。

如果反应不过来，可以代入特殊值，比如 n=1,n=2进去

# 判断一个数组是否为严格单调数组

如何判断序列是否为单增/单减？

很显然我们只要比较每一个数和前一个数是否都满足单调性。如果都满足，根据数学归纳法，那么整个序列也满足。

  • 比较好的做法是维护一个pre节点，每次比较后更新这个pre，这样可以避免特殊处理只有一个节点的情况！！！
    • 比如单增节点，那么初始化pre为负无穷，那么只有一个节点，跟负无穷比也是满足单调性的。
  • 类似于链表的哨兵节点

单减序列判断：

维护一个flag和pre=无穷大

//单减
int pre= Integer.MAX_VALUE;
int[] nums=new int[]{5,4,3,2,1};
boolean flag = true; 
for(var num:nums){
    if(num>=pre) {
        flag=false;
       	pre=num;
        break;
}

# 原地数组去重

26. 删除有序数组中的重复项 (opens new window)

带图演示：https://leetcode.cn/problems/remove-duplicates-from-sorted-array/solutions/728105/shan-chu-pai-xu-shu-zu-zhong-de-zhong-fu-tudo

快慢指针做法

class Solution {
    public int removeDuplicates(int[] nums) {
        int n=nums.length;
        if(n==0) return 0;
        int fast=1,slow=1;
        while(fast<n){
            if(nums[fast]!=nums[fast-1]){
                nums[slow]=nums[fast];
                ++slow;
            }
            ++fast;
        }
        return slow;

    }
}

例题：

567. 字符串的排列 (opens new window)

思路：

注意，能用数组当映射，尽量用数组，Map感觉有时候数据量大了会自己抽风

class Solution {
    public boolean checkInclusion(String s1, String s2) {
        //转换问题，s1长为k，那么s2中如果一段k长的序列，其字母出现次数和种类都和s1的相同，那么一定 存在这样一个排列
        //可以使用一个map统计种类和
        int k = s1.length();
        int n = s2.length();
        int []cnt1=new int[26];
        int []cnt2 = new int[26];
        for(var c:s1.toCharArray()){
            cnt1[c-'a']++;
        }
        for(int r=0;r<n;r++){
            cnt2[s2.charAt(r)-'a']++;
            if(r>=k){
                cnt2[s2.charAt(r-k)-'a']--;
            }
            if(r>=k-1){
                if(equals(cnt1,cnt2)){
                    return true;
                }
            }
        }
        return false;
    }
    public Boolean equals(int []cnt1,int []cnt2){
        for(int i=0;i<26;i++){
            if(cnt1[i]!=cnt2[i]){
                return false;
            }
        }
        return true;
    }
}

# 对象数组

参考本站这部分详解 (opens new window)

Question:对象数组为什么还需要初始化其中的元素?

假设我们定义一个List数组，我们怎么初始化?

private List<Integer>[] array;
1

首先我们理解一下，这个array是一个数字，其元素类型是List<Integer>。所以可以如下初始化

array=new List[n];
//array=new ArrayList[n];
1
2

现在这样是创建了一个List对象数组，其元素都是null，所以还要对元素进行初始化

  • 因为new List[n]是动态初始化，而动态初始化时，元素类型如果是引用类型，那么默认值是null，所以还需要再对元素值进行初始化
 Arrays.setAll(array, e -> new ArrayList<>());
1

如果是基础类型数组比如int [][]array = new int[n][n]，这样也是动态初始化，但是int型默认初始值是0，所有当然不用后面那步给元素初始化
