        int []f= new int [n+5];

        f[0]=0;
        for(int i=0;(int)Math.pow(i,2)<=n;i++){

class Solution {
    public int coinChange(int[] coins, int amount) {
        int []f=new int[amount+1];
        int n=coins.length;
        Arrays.fill(f,10005);
        f[0]=0;
        for(int i=0;i<n;i++){
            for(int c=coins[i];c<=amount;c++){
                f[c]=Math.min(f[c],f[c-coins[i]]+1);
            }
        }
        return f[amount]==10005?-1:f[amount];
    }
}

        Arrays.fill(f,1);
        int ans=0;
        for(int i=0;i<n;i++){

class Solution {
    public boolean canPartition(int[] nums) {
        int s=0;
        for(int num:nums) s+=num;
        int n=nums.length;
        if(s%2==1) return false;
        s/=2;
        boolean []f =new boolean [s+1];
        f[0]=true;
        for(int i=0;i<n;i++){

            }
        }
        return f[s];
    }
}

# 32. 最长有效括号

32. 最长有效括号 (opens new window)

思路：

https://leetcode.cn/problems/longest-valid-parentheses/solutions/314827/shou-hua-tu-jie-zhan-de-xiang-xi-si-lu-by-hyj8

有两种解法，动态规划/栈，但是这道题的动态规划的转移方程很难想，所以这里只记录栈的解法。

直接括号有效的办法为什么不行？

对于括号匹配，我们首先会想到左括号入栈，遇到右括号出栈的方法，判断是否有效，并且同时可以记录当前有效的括号长度。

比如: ()(),遍历这个字符串，每次遇到右括号，那么当前栈顶的左括号出栈，更新答案。

但是显然，最后我们判断都是当前的有效括号，长度都是2。但是答案都是4.

怎么改？

因为我们把匹配的括号都出栈了，所以我们需要一个参照物来辅助求得下标。，也就是栈需要留一个垫底的参照物，用于计算一段连续的有效长度。

可以记录上一个未匹配的右括号的索引(充当参照物)

以)()()为例，遍历到最后一个右括号时，我们记录了上一个未匹配的右括号,下标是0，当前的是4，所以4-0=4.

边界处理：

一开始压入-1

具体算法：

  • 一开始-1入栈，当参照物，处理()()这种情况
  • 遇到左括号，直接入栈
  • 遇到右括号，栈顶直接出栈
    • 如果此时栈不为空，所以这里更新答案， i-st.peek()
      • 这里有两种可能，我们举两种例子
        • (()，上面出栈了一个左括号，还剩一个左括号，此时长度是2-0=2,符合答案
        • (),上面出栈了一个左括号，还剩一个参照物的下标-1, 长度是1-(-1)=2，也符合答案
    • 如果为空，那么此时的右括号入栈，当参照物（保持参照物是上一个未匹配的右括号索引）
      • 比如))()这种情况，要记录的参照物索引就是1，这样更新就是对的
class Solution {
    public int longestValidParentheses(String S) {
        int max=0;
        Deque<Integer>st =new ArrayDeque<>();
        char [] s=S.toCharArray();
        st.push(-1);
        int n=s.length;
        for(int i=0;i<n;i++){
            if(s[i]=='('){
                st.push(i);
            }else{
                st.pop();
                if(st.isEmpty()){
                    st.push(i);
                }else{
                    max=Math.max(max,i-st.peek());
                }
            }
        }
        return max;
    }
}

        int m=grid.length,n=grid[0].length;

        for(int i=0;i<n;i++) f[i][i]=true;

                r=j;
            }
            }
          
        }

# 516最长回文子序列

516. 最长回文子序列 (opens new window)

给你一个字符串 s ，找出其中最长的回文子序列，并返回该序列的长度。

思路1[转换]：

既然是找回文子序列，那么这个子序列从左读和从右读应该是一样的。

所以可以把s反转为s(rev)，求s和s(rev)的最长公共子序列就是s的最长回文子序列(LCS)

思路2[选或不选]

由于是回文子序列，所以我们可以直接看当前字符串最左边和最右边是不是相等

  • 相等可以都选了
  • 不相等产生分支，选其中一个

回溯做法

定义表示从到的最长回文子序列长度(类似LCS)

状态转移如下

  • 如果左右相等，转移到下一个状态，答案加2
  • 如果不相等，则可能是左边不选，右边不选，转移到其中最大的
    • 不会是左右都不选，因为这就是原状态

dfs(i,j)= \begin{cases} dfs(i+1,j-1)+2& \text{s[i]=s[j]}\\ max(dfs(i+1,j),dfs(i,j-1))& \text{s[i]!=s[j]} \end{cases}

递归边界

  • 单个字符就是回文字符串

  • 长度为0，当然是0

    • 比如遇到，会从递归到

入口是

class Solution {
    int [][]vis;
    int n;
    char []s;
    public int longestPalindromeSubseq(String S) {
        s = S.toCharArray();
        n=s.length;
        vis=new int[n][n];
        return dfs(0,n-1);

    }
    public int dfs(int i,int j){
        if(i>j) return 0;
        if(i==j) return 1;
        if(vis[i][j]!=0) return vis[i][j];
        if(s[i]==s[j]) vis[i][j]=dfs(i+1,j-1)+2;//注意是+2
        else{
            vis[i][j]=Math.max(dfs(i+1,j),dfs(i,j-1));
        }
        return vis[i][j];
    }
}

class Solution {
    public int longestPalindromeSubseq(String S) {
        char []s =S.toCharArray();
        int n=s.length;
        int [][]dp =new int [n+1][n+1];
        for(int i=n-1;i>=0;i--){
            dp[i][i]=1;
            for(int j=i+1;j<n;j++){
                if(s[i]==s[j]){
                    dp[i][j]=dp[i+1][j-1]+2;
                }else{
                    dp[i][j]=Math.max(dp[i][j-1],dp[i+1][j]);
                }
            }
        }
        return dp[0][n-1];
    }
}

class Solution {
    public int longestCommonSubsequence(String text1, String text2) {
        char []s=text1.toCharArray();

        char []t =text2.toCharArray();
        int m=s.length;
        int n=t.length;
        int [][]f =new int[m+1][n+1];
        for(int i=0;i<m;i++){

                }
            }
        }
        return f[m][n];
    }
}

class Solution {
    public void nextPermutation(int[] nums) {
        int n=nums.length;
        int i =n-2;
        while(i>=0&&nums[i]>=nums[i+1]){
            i--;
        }
        if(i>=0){
            int j=n-1;
            while(j>=0&&nums[i]>=nums[j]) j--;
            swap(nums,i,j);
        }
        reverse(nums,i+1);
    }

    public void swap(int []nums,int i ,int j){
        int tmp=nums[i];
        nums[i]=nums[j];
        nums[j]=tmp;
    }
    public void reverse(int []nums,int start){
        int l=start,r=nums.length-1;
        while(l<r){
            swap(nums,l,r);
            l++;
            r--;
        }
    }
}

# 287. 寻找重复数

287. 寻找重复数 (opens new window)

思路：

注意数据范围，都在[1,n]之间，那么n+1长度的数组必然至少有一个重复的数，

如果把数组下标看为指针,由于数组的值都在1-n，那么数组的值也一定指向数组中的一个数。由于存在重复数字，那么肯定存在环。

所以可以用快慢指针法找环。

class Solution {
    public int findDuplicate(int[] nums) {
        int slow=nums[0];
        int fast=nums[nums[0]];
        while(slow!=fast){
            slow=nums[slow];
            fast=nums[nums[fast]];
        }
        int x=0;
        while(x!=slow){
            x=nums[x];
            slow=nums[slow];
        }
        return x;
    }
}
