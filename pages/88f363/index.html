# 209. 长度最小的子数组（不定长）

209. 长度最小的子数组 (opens new window)

  • 注意都是正数,所以和有单调性，只会变大

暴力做法

枚举左端点，不断向右拓展(或者枚举右端点，然后不断向左枚举)，直到大于，记录并更新一次答案。

这样做显然是。

# 双指针做法

如何优化呢?我们上面大的做法，没有利用到都是正整数这个性质

我们在下面这个状态时，已经了，所以不会继续向右拓展。按照暴力做法，我们应该枚举2作为左端点，然后再向右拓展。

但是这里我们发现，我们可以缩写左端点而不是从左端点旁边从新枚举。

不断缩写左端点。直到，那么可以更新一次答案，因为题目求的是满足targte的最小长度子数组

更新完后，继续枚举右端点，直接，然后缩写左端点，直到右端点到达边界。

时间复杂度分析: 我们枚举从左到右子数组右端点，是的，然后不断收缩左端点，也是的

第一种写法代码

class Solution {
    public int minSubArrayLen(int target, int[] nums) {
        int n=nums.length;
        int ans = n+1;
        int left=0,r=0;
        int sum=0;
        for( r=0;r<n;r++){//注意是for循环，并且r从零开始，一般这样比较简单，因为不用预先处理第一个窗口
            sum+=nums[r];//枚举右端点，求和
            while(sum-nums[left]>=target){//如果超过target，我们就有可能需要更新答案
                sum-=nums[left];
                left+=1;
            }
            if(sum>=target){//sum>=target才能更新答案
                ans = Math.min(ans,r-left+1);
            }
        }
        return ans==n+1?0:ans;
    }
}

# 713. 乘积小于 K 的子数组

713. 乘积小于 K 的子数组 (opens new window)

上面是从满足要求到不满足要求，也可以是从满足要求到不满足要求

  • 注意都是正数,所以乘积有单调性，只会变大

思路

同样枚举右端点，不断缩短左端点即可。

维护一个乘积，乘上当前元素。如果不满足要求了，就不断缩短左端点，直到

  • 如何计算子数组数目?需要推一个公式
  • 我们当前枚举的是右端点，r是固定的，如果[l,r]是乘积是严格小于k的，那么[l+1,r],.[l+2,r]...[r,r]都是乘积小于k的子数组
  • 这样的数组个数，同样代入特殊值想应该算1个，所以这里应该是要加1的

注意

特殊情况处理

  • k<=1
    • 这里的mul初始化为1，而，所以当时，根据定义我们应该直接返回0，不然会导致下面的循环一直右移出错

代码如下

class Solution {
    public int numSubarrayProductLessThanK(int[] nums, int k) {
        int n=nums.length;
        int mul=1;
        if(k<=1){
            return 0;
        }
        int ans=0;
        int left=0,r=0;
        for( r=0;r<n;r++){
            mul*=nums[r];
            while(mul>=k){
                mul/=nums[left];
                left+=1;
            }
            ans+=r-left+1;
        }
        return ans;
    }
}

# 3. 无重复字符的最长子串(不定长)

3. 无重复字符的最长子串 (opens new window)

思路：

使用一个hash表记录字符和出现次数，

维护一个没有重复字符的子串，每次把当前r指向的字符添加到子串。

因为之前子串是不重复的，所以只可能是当前子串导致有重复字符。

所以此时不断缩小左端点，直到变成一个没有重复字符子串。

  • 注意对应的字符出现次数减1

最后更新答案

代码

class Solution {
    public int lengthOfLongestSubstring(String s) {
        Map<Character,Integer>mp = new HashMap<>();
        int ans=0;
        int left=0,r=0;
        int n=s.length();
        for(r=0;r<n;r++){
            mp.put(s.charAt(r),mp.getOrDefault(s.charAt(r),0)+1);
            while(mp.get(s.charAt(r))>1){
                mp.put(s.charAt(left),mp.get(s.charAt(left))-1);
                left+=1;
            }
            ans = Math.max(r-left+1,ans);
        }   
        return ans;
    }
}

# 理论总结

之所以能使用同向双指针，是因为两个指针之间（窗口）维护的序列具有某种单调性，。比如都是正整数，不重复等等。套路也是一样的

  • 枚举右端点
  • 左右端点之间的序列满足题目要求/不满足要求（单调性）
    • 左右指针之间的这个序列/距离，也可以叫窗口
    • 窗口内序列一般需要满足题目要求，并且不断枚举右端点时会破坏这种要求
      • 这里是难点，很多题目构造窗口符合题目要求/不符合要求比较难转化
      • 可能需要使用逆向思维，牢记正难则反
  • 每次先把当前右端点元素加入到序列，由于之前序列都是单调性的，所有只可能是当前这个元素破坏了单调性，所以此时不断缩小左端点继续保证单调性
  • 更新答案，可能在内部循环内，也可能在外边，具体得看题目的要求
    • 这里常常要推导一下，之间的关系，记住代入特殊值可以快速考虑问题

# 1004. 最大连续1的个数 III

1004. 最大连续1的个数 III (opens new window)

灵神题解:https://leetcode.cn/problems/max-consecutive-ones-iii/solutions/2126631/hua-dong-chuang-kou-yi-ge-shi-pin-jiang-yowmi/

题目中由于存在翻转，我们直接遍历数组，维护一个都是1的窗口好像不太行,所以需要转换一下问题，保证窗口应该是符合题目要求的

思路: 统计窗口内0的个数cnt,则问题转换成的情况下，窗口大小的最大值

  • 这里的单调性就是窗口内0个数小于k
  • 窗口大小公式:
    • 特殊值: r=left,此时窗口大小是1，所以需要加1

代码：

class Solution {
    public int longestOnes(int[] nums, int k) {
        int ans=0;
        int cnt=0;
        int n=nums.length;
        int left=0,r=0;
        for(r=0;r<n;r++){
            cnt+=nums[r]==0?1:0;        
            while(cnt>k){
                cnt-=nums[left]==0?1:0;
                left+=1;
            }
            ans =Math.max(r-left+1,ans);
        }
        return ans;
    }
}

# 1234. 替换子串得到平衡字符串

1234. 替换子串得到平衡字符串 (opens new window)

灵神题解：https://leetcode.cn/problems/replace-the-substring-for-balanced-string/solutions/2108358/tong-xiang-shuang-zhi-zhen-hua-dong-chua-z7tu/

思路:

逆向思维

  • 根据题意，如果在待替换子串之外的任意字符（4种中的一种）出现次数超过，那么无论怎么替换，都不能让这个字符在整个字符串中出现次数为

  • 反过来说，如果待替换子串之外的任意字符的出现次数都不超过，那么就可以通过替换，让称为平衡字符串，即每个字符出现次数均为

所以这里得出

  • 我们依然是枚举右端点，由于窗口之外元素的的序列性质是:任意字符出现次数都不超过
    • 所以窗口其实就是待替换子串
    • 窗口之外是待替换子串之外元素
  • 每次把当前元素

注意： 需要先遍历一次字符串，统计每种字符出现次数，因为这是待替换元素之外的字符

代码:

class Solution {
    public int balancedString(String S) {
        char []s = S.toCharArray();
        int []cnt  =new int[252];
        int n=s.length,m=n/4;
        for(char c: s){
            cnt[c]++;
        }
        if(cnt['Q']<=m&&cnt['W']<=m&&cnt['E']<=m&&cnt['R']<=m){
            return 0;
        }
        int r=0,left=0,ans=n+1;
        for(r=0;r<n;r++){
            --cnt[s[r]];
            while(cnt['Q']<=m&&cnt['W']<=m&&cnt['E']<=m&&cnt['R']<=m){//注意这是窗口之外元素性质
                ans = Math.min(r-left+1,ans);
                ++cnt[s[left]];//窗口缩小，待替换字符外元素就增多
                left+=1;
            }
        }
        return ans;
    }
}

# 1658. 将 x 减到 0 的最小操作数

1658. 将 x 减到 0 的最小操作数 (opens new window)

灵神题解:https://leetcode.cn/problems/minimum-operations-to-reduce-x-to-zero/solutions/2048811/ni-xiang-si-wei-pythonjavacgo-by-endless-b4jt/

思路：

记住正难则反

逆向思维

首先我们看原问题，每次操作可以移除数组最左边或者最右边的元素值，使在最少移除元素个数的情况下，移除元素和为x

所以问题可以转换为，从nums中移除一个最长的子数组，使得剩余元素和为x

  • 由于没有负数，所以可以用滑动窗口
  • 窗口性质就是：最长的子数组，并且其元素和
  • 套路都一样，枚举右端点，缩小左端点

用图来说明，假设sum是nums中元素和

、

特判：

如果整个nums元素和<x，返回0

class Solution {
    public int minOperations(int[] nums, int x) {
        int target=-x;//target就是sum-x
        for(var num:nums){
            target+=num;
        }
        if(target<0){
            return -1;
        }
        int n=nums.length;
        int r=0,left=0,sum=0,ans=-1;
        for(r=0;r<n;r++){
            sum+=nums[r];
            while(sum>target){
                sum-=nums[left];
                left+=1;
            }
            //注意这个别写到循环里面，因为循环条件是>，如果刚好相等就导致没有更新到
            if(sum==target) ans=Math.max(ans,r-left+1);
                //注意ans是最大子数组长度
        }
        //如果ans==1，说明没有sum==target的情况
        //
        return ans==-1?-1:n-ans;
    }
}

# 2962. 统计最大元素出现至少 K 次的子数组

2962. 统计最大元素出现至少 K 次的子数组 (opens new window)

灵神题解：https://leetcode.cn/problems/count-subarrays-where-max-element-appears-at-least-k-times/solutions/2560940/hua-dong-chuang-kou-fu-ti-dan-pythonjava-xvwg/

思路：

  • 注意读题，本题要求的最大元素是整个nums中的最大元素，不是某个子数组中的最大元素！
  • 因为要求的就是子数组，正向思考即可。
  • 窗口内序列性质: 最大元素次数
  • 所以加入当前元素导致最大元素出现次数之后，不断缩小左端点，更新ans
  • 如何更新ans?

所以我们直接在更新完成后

Question:为什么这么做正确?

Answer: 因为L停止的位置是刚好元素出现次数大于k的位置的右边一位，假设我们把L->L-1，那么此时窗口中就有K个最大元素，满足要求。

那么窗口左边的子数组也一定满足要求，而其个数是L-1+1个，所以可以直接加上L

代码:

class Solution {
    public long countSubarrays(int[] nums, int k) {
        int mx = 0;
        for(int x:nums) mx=Math.max(mx,x);
        int cnt=0;
        int left=0,n=nums.length;
        long ans=0;
        for(int r=0;r<n;r++){
            if(nums[r]==mx) cnt++;
            while(cnt==k){
                if(nums[left]==mx) cnt--;
                left+=1;
            }
            ans+=left;
        }
        return ans;
    }
}

思路：

一开始做这道题，我是这个思路

class Solution {
    public int minOperations(int[] nums) {
        int n=nums.length;
        int []f= new int[n];
        List<Integer>g = new ArrayList<>();
        Arrays.sort(nums);
        for(int i=0;i<n;i++){
            int x=nums[i];
            boolean flag=true;
            int j=lowerBound(g,x+1);
            if(g.size()==0){
                g.add(x);
            }else if(j==g.size()){
                if(g.get(j-1)+1==x){
                    g.add(x);
                    f[i]=j+1;
                }else{
                    f[i]=j;
                }
            }else{
                g.set(j,x);
                f[i]=j+1;
            }
        }
        int res=0;
        for(var c:f){
            res=Math.max(res,c);
        }
        return n-res;
        

    }
    public int lowerBound(List<Integer>nums, int target){
        int left=-1;
        int right=nums.size();//开区间
        while(left+1<right){
            int mid = left+(right-left)/2;
            if(nums.get(mid)<target){
                left=mid;
            }else{
                right=mid;
            }
        }
        return right;


    }
}

# 正确思路：排序去重+滑动窗口

虽然不是求严格的LIS，但是正难则反的思想是对的。

灵神题解：https://leetcode.cn/problems/minimum-number-of-operations-to-make-array-continuous/solutions/1005398/on-zuo-fa-by-endlesscheng-l7yi

注意题目的顺序不重要，所以我们给他排下序！！排序之后，我们得到一个递增的数组。

由于我们要求的是严格连续的数组（各个元素不相同），所以应该把重复的元素删掉。

注意题目要求的最终答案是顺序无关的，所以这里删了重复元素就是最优的，帮助我们找到最长的连续数组。

Question：为什么x一定在a[i]中?

Question:窗口性质是什么？

Ans：窗口性质是以为最大值，(left,i)中有最长的严格连续数组长度

为了算出窗口内有多少个点，我们需要知道窗口包含的最左边的点在哪，设这个点是,那么他必须大于等于窗口左边界

我们之前讨论了,如果最大值是,左边界是，现在枚举就是枚举

class Solution {
    public int minOperations(int[] nums) {
        Arrays.sort(nums);
        int n = nums.length;
        int m = 1;
        //快慢指针原地去重
        for (int i = 1; i < n; i++) {
            if (nums[i] != nums[i - 1]) {
                nums[m++] = nums[i]; // 原地去重
            }
        }

        int ans = 0;
        int left = 0;
        for (int i = 0; i < m; i++) {
            while (nums[left] < nums[i] - n + 1) { // nums[left] 不在窗口内
                left++;
            }
            ans = Math.max(ans, i - left + 1);
        }
        return n - ans;
    }
}

# 209 长度最小的子数组

解法：

  • 滑动窗口就是同向双指针，所以我们定义两个指针，left和right
  • 序列要求是大于target
  • 求的是最小长度，所以初始化为Integer.MAX_VALUE,长度=(right-left+1)

所以，可以得到

code

class Solution {
    public int minSubArrayLen(int target, int[] nums) {
        int n=nums.length;
        if(n==0){
            return 0;
        }
        int ans = Integer.MAX_VALUE;//注意我们要的是长度
        int l=0,r=0;
        int sum=0;
        while(r<n){
            sum+=nums[r];
            while(sum>=target){
                ans= Math.min(ans,r-l+1);
                sum-= nums[l];
                l++;
            }
            r++;
        }
        return ans == Integer.MAX_VALUE?0:ans;
    }
}

# lt76 最小覆盖子串

76. 最小覆盖子串 (opens new window)

思路 ：滑动窗口+hash表，注意细节

看到要求满足什么性质的子串，这里就联想到了滑动窗口。 这里求覆盖t中所有字符的s中的最小子串。我们可以用一个哈希表表示 t中所有的字符以及它们的个数，用一个哈希表动态维护窗口中所有的字符以及它们的个数，如果这个动态表中包含 t 的哈希表中的所有字符，并且对应的个数都不小于 t 的哈希表中各个字符的个数，那么当前的窗口是「可行」的。维护一个最小长度的字符串，每次发现可行解就更新最小子串(或者是这个最小子串起始索引和长度)。 窗口怎么移动？ 我们每次移动右指针，如果发现了可行解： 左指针移动到当前第一个不包含可行解的地方(l<r)，然后继续移动r，寻找下一个可行解。 举例 比如当前解是s=ABDCA,t=ABD, 当r=3时，有可行解，那么l移动到B，也就是l=2就又没有可行解了，然后继续移动r。 如何统计是否达到可行解 可以用一个变量tot记录当前已经满足要求的对应字符个数

官方的图解很不错，不过代码太难看

class Solution {
    public String minWindow(String s, String t) {
        HashMap<Character,Integer> need = new HashMap<Character,Integer>();
        HashMap<Character,Integer> windows = new HashMap<>();
        //hash表中只记录与t相关字符，不相关的不用管
        for(char c:t.toCharArray()) need.put(c,need.getOrDefault(c,0)+1);

        int l=0,r=0;
        int tot=0;
        int st=0;//记录最小字符串起始索引位置
        int minLen=Integer.MAX_VALUE;
        while(r<s.length()){
            char c = s.charAt(r);
            r++;
            // 判断取出的字符是否在字串中,是的化更新windows，并判断是否需要更新need
            if(need.containsKey(c)){
                windows.put(c,windows.getOrDefault(c,0)+1);
                if(windows.get(c).equals(need.get(c))){
                    tot++;//更新达到要求的种类数
                }
            }
            while(tot == need.size()){
                if(r-l<minLen){
                    st=l;
                    minLen=r-l;
                }
                char cur = s.charAt(l);
                l++;//左指针指向下一个字符
                //还需要更新windows和对应
                if(need.containsKey(cur)){
                    if(windows.get(cur).equals(need.get(cur))){
                        tot--;//更新相等的种类数
                    }
                    windows.put(cur,windows.get(cur)-1);
                }
            }
        }
        return minLen==Integer.MAX_VALUE?"":s.substring(st,st+minLen);
    }
}

# 总结

  • 应用场景：

    • 子数组，字串或者把题目反向思考过来，跟子数组，字串有关，可能就可以考虑滑动窗口/同向双指针
      • 还可以有附加条件，比如数组元素都是正数，0或者1这种点

  • 如果滑动窗口是定长的，比如说​，要注意第一次的情况，以及循环怎么写

    • 如果没有预处理，我们一般是写到循环里，判断时才更新答案，但是这样写有个好处就是不用特殊处理，[少出错]，推荐
    • 注意循环中是否需要判断第一次循环，往往第一次循环不满足我们写的循环体，比如[2090]

  • 很多题不是一眼就能看出滑动窗口，而是我们先分析问题，最后发现滑动窗口可以做。

    • 比如要先排序，先转化问题等等
