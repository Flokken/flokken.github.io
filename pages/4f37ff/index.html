# 单调栈

单调栈 (opens new window)

灵神单调栈讲解 (opens new window)

单调栈，也就是栈内元素具有单调性（一般是栈底到栈顶）。如果是栈底元素到栈顶元素单调递增，则是递增栈；如果是栈底元素到栈顶元素单调递减，就是递减栈。

tips:相等元素不是单调哦，一般相等元素时要出栈\入栈

这样不太直观，我们直接看图

# 739. 每日温度

739. 每日温度 (opens new window)

给定一个整数数组 temperatures ，表示每天的温度，返回一个数组 answer ，其中 answer[i] 是指对于第 i 天，下一个更高温度出现在几天后。如果气温在这之后都不会升高，请在该位置用 0 来代替。

假设从左往右遍历数组，对于每一个数，我们从这个点往右找第一个比他大的数，那么最坏情况下，所有元素相等，那么需要的时间。

# 从右到左的做法

模拟一下，一开始是6，然后是3，此时3已经找到右边最大值，所以可以记录答案；我们先存进去，2也存进去，记录答案

6,3,2；然后遇到5，此时单调性被打破，3出栈，2出栈，然后5记录答案并且进栈

因为6不能直接出栈，不然前面的5就记录不了其右边最大的数了

此时我们栈里存的是已经找到右边最大值的数

假设我们从右往左遍历这个数组，同时维护一个栈，保证栈内元素递减（递减栈）。所以一旦当前元素比栈顶大，栈顶元素就要出栈，直到栈顶元素比当前元素大。

这样当元素出栈的时的右边的第一个遇到的最大元素就是栈顶元素，栈中记录元素是还没有找到答案的数。模拟如下:

最核心的是及时去除无用元素，保持栈中数据有序

我的理解: 这里是找更大元素，我们维护一个单减栈，一旦有更大的元素出现，那么之前的小的元素都被挡住了(比如上面5后面的2,3)，所以这些都成了无用元素，就要及时丢掉。

class Solution {
    public int[] dailyTemperatures(int[] temperatures) {
        int n=temperatures.length;
        int [] ans=new int [n];
        Deque<Integer>st = new ArrayDeque<>();
        for(int i=n-1;i>=0;i--){
            while(!st.isEmpty()&&temperatures[i]>=temperatures[st.peek()]){//因为找到是右边第一个最新的，而我们这里是逆序，所以相等的数记录最左边的，是>=
                st.poll();
            }
            if(!st.isEmpty()){
                ans[i]=st.peek()-i;
            }
            st.push(i);
        }
        return ans;
    }
}

# 从左到右的做法

我们从左到右思考一下，首先是1，然后是4,1的右边最大值已经找到，出栈记录答案

然后4,3；遇到5，此时4,3出栈记录答案

此时我们存的是还没有找到最大值的数

class Solution {
    public int[] dailyTemperatures(int[] temperatures) {
        int n=temperatures.length;
        int [] ans=new int [n];
        Deque<Integer>st = new ArrayDeque<>();
        for(int i =0;i<n;i++){
            while(!st.isEmpty()&&temperatures[i]>temperatures[st.peek()]){//这里严格大于才出栈，更新答案
                int t=st.poll();
                ans[t]=i-t;
            }
            st.push(i);
        }
        return ans;
    }
}

# 42.接雨水

42. 接雨水 (opens new window)

单调栈解接雨水 (opens new window)

之前双指针或者前后缀做法，我们是竖着看的，一个点看成一个水桶。

从左向右遍历

这道题也可以横着看。我们假设有一个栈，5 2 1 0 然后遇到4，那么这一块的面积是

1 0 4 ,对应1*1，然后0出栈，记录答案

2 0 4 对应 1*2，然后1出栈，记录答案

5 2 4 对应2*3，然后2出栈，记录但

此时 4进栈。

我们需要三个下标，栈顶下标，当前柱子下标，栈顶下面那个元素下标。

核心:找上一个更大元素，找到过程种填坑。

假设L,R分别是左右两根柱子下标，h1（左）,h2（右）,h3（栈顶下面那根）是三根柱子高度

$ (R-L-1)*(min(h1,h2)-h3)$

对于重复的，我们出栈原来的，记录后面那个，这里很细节

模拟如下:

序列是 5 2 1 0 4，从左到右遍历，单减栈。

那么一开始一直入栈到 5 2 1 0，直到遇到4，开始出栈，并且更新总面积ans

  • 对于0， 结果肯定是0，ans+=0; 0出栈
  • 对于1 ，，ans+=1，1出栈
  • 对于2，，ans+=2,2出栈
  • 对于5，,ans+=6,5出栈
  • 然后4入栈...........

细节：遇到与栈顶相等的元素，我们也会将栈顶出栈处理，避免重复计算，所以这里是

Question:为什么这样做是正确的，我们只记录最后一个相等元素的下标正确？

Ans：因为相等元素只利用到其高度，并不影响到计算。举例模拟

序列4 1 1 6, 下标是0 1 2 3

栈4 1 ，然后遇到1 ，出栈，算出来是0，不影响答案

遇到5,1出栈， 此时hegiht[m]=1. L=0,R=3

ans=ans +(3-0-1)*min(6,4)-1=6。这正是图中那块的面积，所以这里是大于等于。

最终代码实现

牢牢记住:及时去掉无用数据，保证栈中数据有序

class Solution {
    public int trap(int[] height) {
        int n=height.length;
        Deque<Integer>st = new ArrayDeque<>();
        int ans=0;
        for(int i=0;i<n;i++){
            while(!st.isEmpty()&&height[i]>=height[st.peek()]){
                int m=st.poll();//中间那根
                if(st.isEmpty()) break;//没有三根柱子，直接跳出
                int L=st.peek();//最左边那根
                int R=i;//当前柱子，也是最右边那根
                ans += (R-L-1)*(Math.min(height[L],height[R])-height[m]);
            }
            st.push(i);
        }
        return ans;
    }
}

# 知识总结

  • 其实无论是从左到右还是从右到左，我们都一样利用了单调性，
  • 实质只是看待一个问题的不同角度
    • 相应的进栈和更新答案时机也变了

知识点核心如下：

单调栈中也可以有重复元素，具体看题目要求

  • 先进后出

    • 记录的数据加在最上面(栈顶)
    • 丢掉数据也从最上面开始(栈顶)

  • 单减栈单调性

    • 记录之前会把所有​的数据丢掉
    • 所以不可能出现下面小上面大的情况
    • 单调性是我们需要利用的，所以有这种性质的问题就可以用单调栈试试，不要死记硬背

  • 考虑使用单调栈的情况

    • 上一个更大元素
    • 下一个更大元素

  • 单调栈16字总结:及时去掉无用数据，保证栈中数据有序

  • 我总结的(还不一定对): 一般来说： 找更小元素使用单增栈，更大元素使用单减栈

# 1475. 商品折扣后的最终价格

1475. 商品折扣后的最终价格 (opens new window)

思路：

类似于每日温度，只不过我们这里需要的是比当前元素小的右边的第一个下标，所以是维护一个单增栈。

注意：

  • 有可能有些天数后面找不到更小的，应该把他们设置为对应的​。
  • 等于也满足要求，所以也出栈
class Solution {
    public int[] finalPrices(int[] prices) {
        int n=prices.length;
        Deque<Integer>st =new ArrayDeque<>();
        int []ans =new int[n];
        for(int i=0;i<n;i++){
            while(!st.isEmpty()&&prices[i]<=prices[st.peek()]){
                int t=st.poll();
                ans[t]=prices[t]-prices[i];
            }
            st.push(i);
        }
        while(!st.isEmpty()){
            int t=st.poll();
            ans[t]=prices[t];
        }
        return ans;
    }
}

# 901. 股票价格跨度

901. 股票价格跨度 (opens new window)

思路：

我们要找从今天往回找的最长连续低于今天的天数。那么可以转化为找当前​​的上一个最大元素，所以可以用单调栈。

  • 注意这里需要每天输出当前跨度

Question:维护单增栈还是单减栈呢？

Ans：思考当前元素和栈顶元素关系，如果当前元素比栈顶大，栈顶元素都用不上了，那么就应该让栈中元素出栈，也就是及时去除无用数据，保证栈中元素有序。所以这是单减栈。

如果当前元素比栈顶元素大，那么之前的栈顶元素就无用了，应该出栈。所以这里应该维护了一个单减栈

而；i是今天，st.peek()就是第一个大于今天的下标

  • 特殊情况，如果当前元素比所有元素都大，那么栈为空，需要特殊处理。此时，也就是当前序列长度
    • 也可以事先向栈里添加一个元素，其下标为-1，一样的效果

注意：这里指的是从今天开始的连续天数，不是从今天开始的不算，理解题目很重要

比如[20,20,20,80,70]对应是[1,2,3,4,1]，最后70对应的是1而不是3，因为要求这个天数是从今天往回数

class StockSpanner {
    Deque<Integer>st;
    int i;
    int []prices;
    public StockSpanner() {
        st=new ArrayDeque<>();
        prices = new int[10005];
        i=-1;//这样第一次插入元素才是对应到0
    }
    
    public int next(int price) {
        i++;
        prices[i]=price;
        int ans=0;
        while(!st.isEmpty()&&prices[i]>=prices[st.peek()]){
            st.pop();
        }
        if(st.isEmpty()){
            ans=i+1;//如果当前元素比栈中所有元素大，那么其就应该是序列长度（下标加1）
        }else{
            ans=i-st.peek();
        }
        st.push(i);
        return ans;
    }
}

# 1019. 链表中的下一个更大节点

1019. 链表中的下一个更大节点 (opens new window)

思路：

我们要找当前节点右边第一个更大元素。我们从左到右遍历元素，如果当前元素比栈顶大，那么当前元素就是栈顶的下一个更大值。然后栈顶应该出栈，因为找到了。所以这里应该是单减栈。

注意：

  • 这里是严格大于，所以应该是​时，才出栈栈顶更新其答案。

  • 如果是没有当前元素没有下一个更大元素，设置为0，初始化就是0，这里好处理

    • 如果是其他值，应该出栈剩余元素，设置他们

  • 这是链表，我们需要之前的值，所以可以维护一个数组和下标存之前的值。

class Solution {
    public int[] nextLargerNodes(ListNode head) {
        int []nums = new int[10005];
        int i=-1;
        int []ans= new int[10005];
        Deque<Integer>st =new ArrayDeque<>();
        while(head!=null){
            nums[++i]=head.val;
            while(!st.isEmpty()&&nums[i]>nums[st.peek()]){
                int t=st.poll();
                ans[t]=nums[i];
            }
            st.push(i);
            head=head.next;
        }

        int []res = new int[i+1];
        for(int j=0;j<i;j++){
            res[j]=ans[j];
        }
        return res;
    }
}

# 1944. 队列中可以看到的人数（重要！）

1944. 队列中可以看到的人数 (opens new window)

思路：

注意这里的问题是其能看到的人，可能存在矮子被挡住的情况。

这里可以类比每日温度那道题，我们从左往右维护遍历，当前元素找到了下一个更大元素，那么当前元素可以更新答案了。这个找下一个更大元素的问题应该是单调栈。

单增还是单减？

如果栈顶元素小于当前元素，那么当前元素出栈，更新其答案，所以这是单减栈。

怎么更新？

10没思路，20分钟没写出来，直接看题解，这里浪费了一个多小时，太亏了，

https://leetcode.cn/problems/number-of-visible-people-in-a-queue/solutions/2591558/dan-diao-zhan-de-ben-zhi-ji-shi-qu-diao-8tp3s

这里需要逆向思考，也很需要经验，不能死耗！

具体来说：

我们从右往左遍历，我们比较栈顶与当前元素，栈顶元素小于当前元素时，我们就不断出栈，这些就是比当前元素小的元素，所以都能看到。

而且注意出栈结束后如果栈不为空，说明第个人还可以看到一个比他栈顶，；

这里很妙，我们从右到左遍历维护单减栈，那么从左到右就都是看到一个单增的序列，这都是我们能看到的，然后如果栈不为空，那么可以再看到这个比当前元素高所以没出栈的元素，所以还加1.

这些都是

所以这是找下一个更大元素，单减栈。

并且不需要下标，所以可以直接入栈高度，而不是下标

  • 相等元素呢？
    • 我们可以看特例来思考，比如[2,2,2,2]。那么都是0，所以这里应该是。是不更新答案
class Solution {
    public int[] canSeePersonsCount(int[] nums) {
        int n=nums.length;
        int []ans =new int[n];
        Deque<Integer>st = new ArrayDeque<>();
        for(int i=n-1;i>=0;i--){
            while(!st.isEmpty()&&st.peek()<nums[i]){
                st.pop();
                ans[i]++;
            }
            if(!st.isEmpty()){//如果栈不为空，所以当前元素还能看见这个栈顶
                ans[i]++;
            }
            st.push(nums[i]);
        }
        return ans;
    }
}

# 84. 柱状图中最大的矩形(前后缀分解+单调栈)

84. 柱状图中最大的矩形 (opens new window)

灵神题解 (opens new window)

思路？

首先对于有最大矩阵的面积，他的高度一定是某个，这是显然的

反证法，比如例子中，我们假设最大矩阵最大高度是4，那么这不对，因为继续增高到5面积会更大

所以可以假设是矩阵高度。然后枚举每个height，乘以其宽度，肯定就能从所有结果中得到最大结果。

那么此时的矩阵宽度是多少?我们需要知道。

这种固定一个变量来思考问题的思想很重要

这是显然的，由于我们的高度已经选定，所以其左边和右边的柱子高度不能小于。

所以可以找到左边的更近的更小元素和右边的更近的更小元素

  • 在左侧小于的最近元素的下标
  • 在右侧小于的最近元素下标​

上一个更小元素和下一个更小元素，所以肯定是单调栈

那么宽度就是。也就是开区间

以上图为例，当前，，其left就是1（1那根柱子），right就是4（2那根组织）

所以面积是

所以我们枚举，维护矩阵最大值即可

本质还是枚举，只是我们对问题进行了分析，枚举起来更方便

而单调栈可以帮助我们快速找到和

并且我们需要两个辅助数组来存储对应的和。然后统一计算面积，取最大值

注意

  • 下标从0开始到n-1，所以左边界是-1，右边界是​。因为我们利用的是开区间

    • 这里处理很细节，不然[2,2,2,2]这种数组就会处理出错

  • 求更小元素，那么是单增栈，首先看左边，从左到右遍历即可

    • 还是​，我们比较当前元素和栈顶，当前元素如果比栈顶大，那么栈顶就是当前元素左边第一个比他小的。由于是单增栈，所以我们栈中元素的左边最小元素我们已经知道了。此时可以直接去掉，直到当前元素找到其左边的第一个小于的元素。

      • 举例，比如[1,3,3,4]，

        • 遇到1，，栈:[1]
        • 遇到3，3入栈，,栈:[1,3]
        • 遇到3，3入栈，此时栈顶，出栈，然后到[1]，那么栈顶元素，left[2]=0
        • 遇到4,4入栈，此时​，所以left[4]=0

      • 所以我们为了找到左边第一个当前元素的元素，应该这里是，需要把​的元素出栈

  • 然后看右边最小元素，

    • 从右往左的单增栈，遍历到时,
      • 比如当前元素时3，序列是[1,2,3,5,4],对于3来说，其看到的就是4,5这样的单减序列。
        • 所以仍然是把比栈顶的元素大和相等的元素都出栈，直到遇到第一个右边更小元素
class Solution {
    public int largestRectangleArea(int[] heights) {
        Deque<Integer>st = new ArrayDeque<>();
        int n=heights.length;
        //这里分别用两个数组记录i左边和右边的第一小元素
        int []left=new int[n];
        int []right = new int[n];
        for(int i=0;i<n;i++){
            while(!st.isEmpty()&&heights[i]<=heights[st.peek()]){
                st.poll();
            }
            if(!st.isEmpty()){
                left[i]=st.peek();
            }else{
                left[i]=-1;//一直到边界都没有比heights[i]严格更小的元素
            }
            st.push(i);
        }
        st.clear();
        for(int i=n-1;i>=0;i--){
            while(!st.isEmpty()&&heights[i]<=heights[st.peek()]){
                st.poll();
            }
            if(!st.isEmpty()){
                right[i]=st.peek();
            }else{
                right[i]=n;//一直到边界都没有比heights[i]严格更小的元素
            }
            st.push(i);
        }
        int ans=0;
        for(int i=0;i<n;i++){
            ans=Math.max((right[i]-left[i]-1)*heights[i],ans);
        }
        return ans;
    }
}

class Solution {
    public int maximumScore(int[] nums, int k) {
        int n=nums.length;
        int []right = new int [n];
        int []left = new int [n];
        Deque<Integer>st = new ArrayDeque<>();
        int ans=0;
        for(int i=0;i<n;i++){
            while(!st.isEmpty()&&nums[i]<=nums[st.peek()]){
                st.poll();
            }
            left[i]=st.isEmpty()?-1:st.peek();
            st.push(i);
        }
        st.clear();
        for(int i=n-1;i>=0;i--){
            while(!st.isEmpty()&&nums[i]<=nums[st.peek()]){
                st.poll();
            }
            right[i]=st.isEmpty()?n:st.peek();
            st.push(i);
        }
        for(int i=0;i<n;i++){
            if(k>left[i]&&k<right[i]){
                ans=Math.max(ans,(right[i]-left[i]-1)*nums[i]);
            }
        }
        return ans;
    }
}

# 总结

  • 单调栈实际上只有四种类型，找更大/更小元素，然后从左到右和从右到左。所以是四种

    • 更大元素肯定是单减栈
    • 更小元素肯定是单增栈
    • 是还是?
      • 我们看题目的具体要求，比如当前元素左边第一个小于，维护的单增栈，那么是，因为要把​的也要出栈，然后才能找到
      • 可以举特例比如[3,3,3]这种特殊的例子来判断

  • 细节有

    • 当前元素和栈顶元素什么关系？和问题什么关系？

    • 如果栈中最后有剩余元素怎么办？

    • 假设从左到右，我们找当前元素左边的更小元素，如果其左边都比当前元素大怎么办呢，比如[4,4,3]，对于3来说就是这个情况

      • 这种情况需要根据题目特殊处理，比如84题，开区间左边处理为-1，右边则是n

    • 什么时候更新答案？根据题目来看

      • 可以是在循环体内更新每个出栈元素
      • 也可以是​​循环之外，也就是出栈结束后，更新当前元素

    • 这些细节怎么搞清楚？可以多举例子，特例辅助思考
