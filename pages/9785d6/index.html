# 122. 买卖股票的最佳时机 II

122. 买卖股票的最佳时机 II (opens new window)

prices = [7,1,5,3,6,4]
1

思路：

对于这种选或者不选的问题，我们一般从两侧开始考虑，而为了方便转化为递推，我们一般从末尾开始考虑。

最后一天选/不选？

关键点：天数，是否持有股票（最多持有一个）

# 回溯

回溯三问：

  • 子问题？从第天到第天结束时，持有/未持有股票的最大利润。
  • 当前操作？下图状态机转换
  • 下一个子问题？到第​天结束时，持有/未持有股票的最大利润

当前操作：

状态定义：

  • 定义表示到第天结束时，未持有股票的最大利润。
  • 定义表示到第​天结束时，持有股票的最大利润。

注意我们并没有把持有那只股票加入状态定义，只讨论是否持有股票

注意根据状态机的关系，由于第天的结束就是第天的开始,

  • 所以第天结束时的利润就是第天开始时的最大利润

  • 注意有两个状态,x=0/1

转移方程：

注意要获利一定是之前买了，后面卖，当天买卖没有意义

  • ​
    • 如果这天结束后没有持有股票，那么有两种情况。
    • 直接从上一个没持有的状态转移过来
    • 本来持有，但是卖了
  • ​
    • 如果这天结束后持有股票，那么有两种情况。
    • 直接从上一个持有的状态转移过来
    • 本来没持有，买了今天的股票

递归边界：

,-1天结束，第0天开始，未持有股票肯定是0

，-1天结束，第0天开始不可能持有股票，因此初始化为负无穷

Question：为什么不能把0当做边界呢？

Ans：因为数组元素都是从0开始的，如果以0为边界，我们可能需要特判空数组的情况。为了少写特判，负数显然更合适

递归入口：

最后一天持有股票是没有意义的，肯定是不如的，所以只考虑卖出的情况即可

class Solution {
    int [][]vis;
    int []prices;
    public int maxProfit(int[] prices) {
        int n=prices.length;
        this.prices=prices;
        vis = new int[n][2];
        for(int i=0;i<n;i++){
            Arrays.fill(vis[i],-1);
        }
        return dfs(n-1,0);
    }
    public int dfs(int i, int j){
        if(i<0){
            return j==0?0:Integer.MIN_VALUE;
        }
        if(vis[i][j]!=-1) return vis[i][j];
        if(j==0){
            return vis[i][j]=Math.max(dfs(i-1,0),dfs(i-1,1)+prices[i]);
        }
        return vis[i][j]=Math.max(dfs(i-1,1),dfs(i-1,0)-prices[i]);
    }
}

class Solution {
    int [][][]vis;
    int []prices;
    public int maxProfit(int k,int[] prices) {
        int n=prices.length;
        this.prices=prices;
        vis = new int[n][k+1][2];

        for(int i=0;i<n;i++){
            for(int j=0;j<=k;j++)
                Arrays.fill(vis[i][j],-1);
        }
        return dfs(n-1,k,0);
    }
    public int dfs(int i, int k,int j){
        if(k<0){
            return Integer.MIN_VALUE/2;
        }
        if(i<0){
            return j==0?0:Integer.MIN_VALUE/2;
        }
        if(vis[i][k][j]!=-1) return vis[i][k][j];
        if(j==0){
            return vis[i][k][j]=Math.max(dfs(i-1,k,0),dfs(i-1,k,1)+prices[i]);
        }
        return vis[i][k][j]=Math.max(dfs(i-1,k,1),dfs(i-1,k-1,0)-prices[i]);
    }
}

class Solution {

        int n=prices.length;

思路：

class Solution {
    public int maxProfit(int[] prices, int fee) {
        int n=prices.length;
        int [][]f = new int[n+1][2];
        f[0][0]=0;
        f[0][1]=Integer.MIN_VALUE/2;
        for(int i=0;i<n;i++){

            f[i+1][1]=Math.max(f[i][1],f[i][0]-prices[i]);
        }
        return f[n][0];
    }
}

# 1911. 最大子序列交替和

1911. 最大子序列交替和 (opens new window)

10分钟没确定思路，看灵神题解了：https://leetcode.cn/problems/maximum-alternating-subsequence-sum/solutions/846375/dong-tai-gui-hua-by-endlesscheng-d92a

思路：

感觉比较复杂，我们类比着股票最佳时机的方法来分析。

启发思路：

子序列问题，核心还是选/不选

假设求长为4的数组[4,2,5,3]最大的交替和，那么如果前4个元素的最大交替和就是 前3个元素最大交替加上个第四个的值

  • 可以是加上第4个元素
  • 可以是减去第4个元素
  • 可以加上0，代表没有选择第4个元素

关键点在于，子序列长度是奇数还是偶数，

回溯三问：

表示前​​​​个元素中最大交替和子序列长为偶数的最大值

  • 最后一个元素下标为奇数

表示前​个元素中最大交替和子序列长为奇数的最大值

  • 最后一个元素下标为偶数

Question：为什么不像以结尾的子序列的最大值这样来定义呢？

Ans：这样定义没有重叠子问题，因为​结尾并不能决定什么，决定长度的是子序列中元素的下标

而LIS问题，因为递增是与元素大小有关的，所以才会以结尾来定义状态

对于第个元素，有选/不选两种决策

class Solution {
    public long maxAlternatingSum(int[] nums) {
        int n=nums.length;
        long [][]f=new long [n+1][2];
        f[0][0]=0L;
        f[0][1]=Long.MIN_VALUE/2;
        for(int i=0;i<n;i++){
            f[i+1][0]=Math.max(f[i][0],f[i][1]-(long)nums[i]);
            f[i+1][1]=Math.max(f[i][1],f[i][0]+(long)nums[i]);
        }
        return f[n][1];

    }
}

空间优化：

循环时，我们只用到和两个状态，所以可以用两个变量优化空间

class Solution {
    public long maxAlternatingSum(int[] nums) {
        int n=nums.length;
        long f0=0L;
        long f1=Long.MIN_VALUE/2;
        for(int i=0;i<n;i++){
            long newf0=Math.max(f0,f1-(long)nums[i]);
            f1=Math.max(f1,f0+(long)nums[i]);
            f0=newf0;
        }
        return f1;//对应f[n][1];

    }
}
