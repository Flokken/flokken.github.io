# 组合型回溯

# 77. 组合

77. 组合 (opens new window)

回顾 从[3,2,1]中得到所有子集的搜索树

我们可以发现

从个数中选​个数的组合，可以看成长度固定的子集

组合问题可以做一些额外优化，比如上面，如果选了2，后面只能选1，那么此时是不需要递归我们就知道答案了

设长为，那么还需要选取个数

假设是倒序枚举，也就是从n开始枚举

所以当前需要从这个数中选数，如果，那么最后必然无法选出个数，不需要继续递归

这是一种剪枝技巧

如果是正序枚举，也就是从0开始，一样的当前是第个数，那么还剩余个数（包括）

所以不等式是，则无法选出个数，不需要继续递归，这样写不等式更复杂

上面的搜索树思路也是上节说的从答案的角度想，每次都要选一个

# 从答案角度

从答案角度就是枚举选哪个

回溯三问

  • 当前操作？枚举选哪个数
  • 子问题？从[1,n]中选出k个数加入组合
  • 下一个子问题？从[1,j-1]中选出一个数k-1个数加入组合
    • 因为第个数已经选了，所以这里是

边界条件

  • k==path.size()

剪枝

  • d=k-m, d>i
class Solution {
    List<Integer>path;
    int n,k;
    List<List<Integer>>ans;
    public List<List<Integer>> combine(int n, int k) {
        path =new ArrayList<>();
        ans = new ArrayList<>();
        this.n=n;
        this.k=k;
        dfs(n);
        return ans;
    }
    public void dfs(int i){
        int d=k-path.size();
        if(k-path.size()>i){
            return;
        }
        if(path.size()==k) {
            ans.add(new ArrayList(path));
            return ;
        }
        for(int j=i;j>=1;j--){
            path.add(j);
            dfs(j-1);
            path.remove(path.size()-1);
        }
    }
}

# 216. 组合总和 II

216. 组合总和 III (opens new window)

这里和上一道题区别是加了一个限制条件，所有数相加之和为,导致

  • 更新答案时需要判断是不是和为n
  • 可以多一个剪枝，假设当前和已经大于，那么剪掉
    • 为了更方便，一般反过来写，可以一开始设置为n，每加入数就减小，当，剪枝
  • 对称的想，还有一个剪枝，那就是如果当前剩余数字即使都选最大的，和也小于
    • 例如，还需要选个数，如果现在的，那么可以直接剪枝
    • ,等差数列求和

# 答案角度

这里回溯三问和上一题一样,只需要略微修改剪枝和更新即可

剪枝

  • t>0||t<(2*i-d+1)*d/2

更新答案

  • 需要判断和是否为n
class Solution {
    List<Integer>path;
    int n,k;
    List<List<Integer>>ans;
    public List<List<Integer>> combinationSum3(int k, int n) {
        path =new ArrayList<>();
        ans = new ArrayList<>();
        this.n=n;
        this.k=k;
        dfs(Math.min(n,9),n);//注意只允许使用数字1-9
        return ans;
    }
      public void dfs(int i,int t){
        int d=k-path.size();
        if(k-path.size()>i){
            return;
        }
        if(t<0||t>(2*i-d+1)*d/2) return;
        if(path.size()==k) {
            ans.add(new ArrayList(path));
            return ;
        }
        for(int j=i;j>=1;j--){
            path.add(j);
            dfs(j-1,t-j);
            path.remove(path.size()-1);
        }
    }
}

# 22 括号生成

22. 括号生成 (opens new window)

问题可以转换为从个位置中选个左括号，所以此时选等价于添加左括号，不选等价于添加右括号

约束

  • 左括号不能超过，达到n之后只能选右括号
  • 右括号个数不能超过左括号个数
    • 两者个数相等时，下一个只能选左括号

# 从输入角度

回溯三问

  • 当前操作？枚举是左括号/右括号
  • 子问题?构造字符串的位置
  • 下一个子问题？构造字符串的位置

如何做呢？

字符串的题一般都需要想想怎么实现

首先需要用记录左括号个数，方便判断(这里对编程实现很重要！)

  • 需要选个左括号，那么只要，就可以选左括号
  • 右括号需要个，如果右括号少于左括号个数，那么只要，就可以选右括号
    • 因为状态含义是已经构造了前个，其中有左括号个，那么就是右括号
    • 少于这个条件可以更新右括号个数到左括号个数而不会超过

转移

$dfs(i)->dfs(i+1,open+1) 或者 dfs(i+1,open) $

边界条件

  • ，表示已经构造前n个
    • 因为下标是0到n-1
class Solution {
    char []path ;//用数组可以直接覆盖，避免还原状态
    List<String> ans;
    int n;
    //也可以用stringBuilder，来append和remove
    public List<String> generateParenthesis(int n) {
        path=new char[2*n];//长度是一定的，2*n
        ans = new ArrayList<>();
        this.n=n;
        dfs(0,0);
        return ans;
    }
    public void dfs(int i,int open){
        if(i==2*n){
            ans.add(new String(path));
            return ;
        }
        if(open<n){
            path[i]='(';
            dfs(i+1,open+1);
        }

        if(i-open<open){
            path[i]=')';
            dfs(i+1,open);
        }
    }
}

# 总结技巧

  • 如果需要判断状态的和为，那么一样反过来，每次减去当前的选择，最后根据剩余的数，来判断
    • 这样写比正着写方便很多，正难则反！
