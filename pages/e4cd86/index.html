# 516最长回文子序列

516. 最长回文子序列 (opens new window)

给你一个字符串 s ，找出其中最长的回文子序列，并返回该序列的长度。

子序列定义为：不改变剩余字符顺序的情况下，删除某些字符或者不删除任何字符形成的一个序列。

# 思路1[转换]：

既然是找回文子序列，那么这个子序列从左读和从右读应该是一样的。

所以可以把s反转为s(rev)，求s和s(rev)的最长公共子序列就是s的最长回文子序列(LCS)

# 思路2[选或不选]

我们可以从两侧向内不断缩写规模。

由于是回文子序列，所以我们可以直接看当前字符串最左边和最右边是不是相等

  • 相等可以都选了
  • 不相等产生分支，选其中一个
# 递归做法(记忆化搜索)

定义表示从到的最长回文子序列长度(类似LCS)

根据上面的分析，状态转移如下

  • 如果左右相等，转移到下一个状态，答案加2
  • 如果不相等，则可能是左边不选，右边不选，转移到其中最大的
    • 不会是左右都不选，因为这就是原状态

dfs(i,j)= \begin{cases} dfs(i+1,j-1)+2& \text{s[i]=s[j]}\\ max(dfs(i+1,j),dfs(i,j-1))& \text{s[i]!=s[j]} \end{cases}

递归边界

  • 单个字符就是回文字符串

  • 长度为0，当然是0

    • 比如遇到，会从递归到

时空复杂度

由于状态有个，每个状态计算时间为，所以时间和空间复杂度都是

注意记忆化其实就是起到了剪枝的作用，因为已经递归过的再递归一次是浪费时间

class Solution {
    int [][]vis;
    String s;
    public int longestPalindromeSubseq(String S) {
        int n=S.length();
        s=S;
        vis = new int[n][n];
        for(int i=0;i<n;i++){
            Arrays.fill(vis[i],-1);
        }
        return dfs(0,n-1);
    }
    public int dfs(int i, int j){
        if(i==j){
            return 1;
        }
        if(i>j) return 0;
        if(vis[i][j]!=-1) return vis[i][j];
        if(s.charAt(i)==s.charAt(j)){
            return vis[i][j]=dfs(i+1,j-1)+2;//注意是加2
        }else{
            return vis[i][j]=Math.max(dfs(i+1,j),dfs(i,j-1));
        }  
    }
    
}

# 翻译成递推做法

把dfs()这个函数换成f数组就行了

f(i,j)= \begin{cases} 0&i>j \\ 1&i==j\\ f[i+1,j-1]+2& \text{s[i]=s[j]}\\ max(f[i+1,j],f[i,j-1])& \text{s[i]!=s[j]} \end{cases}

还需要注意一下循环顺序

  • 从转移过来，0所以​需要倒序枚举

    • 这是显然是，如果正序枚举，我们计算是需要，但是这是​还没被更新，所以要倒序枚举

  • 从转移过来，所以​是正序枚举

    • 首先，对于第二位，肯定需要先固定第一维，在都是的情况下，从​​​而来，所以需要正序枚举

还有一个点：

区间DP肯定也是从小问题到大问题

  • 对于区间DP，这里j是从i+1到n才能表示从小区间到大区间

Question:为什么这里也有-1的操作，但是不用替换？

Ans：因为我们这里i一定小于n,j一定大于0，所以不会越界

边界：

循环时，是逆序，时正序

，应该是边界,这些都应该理解

答案是

代码如下:

class Solution {
    public int longestPalindromeSubseq(String s) {
        int n=s.length();
        int [][]f= new int[n][n];
        for(int i=0;i<n;i++){
            f[i][i]=1;
        }
        for(int i=n-1;i>=0;i--){
            for(int j=i+1;j<n;j++){//注意这里
                if(s.charAt(i)==s.charAt(j)){
                    f[i][j]=f[i+1][j-1]+2;
                }else{
                    f[i][j]=Math.max(f[i+1][j],f[i][j-1]);
                }
            }
        }
        return f[0][n-1];
    }
}

# 递归做法(记忆化搜索)

定义表示从到的这个多边形的最低得分，实际是就枚举上面我们划分的子问题

递归边界

两个顶点不能组成三角形，所以是0

递归入口

代码如下

class Solution {
    int [][] vis;
    public int minScoreTriangulation(int[] values) {
        int n=values.length;
        vis = new int[n][n];
        return dfs(0,n-1,values);
    }
    public int dfs(int i,int j,int [] v){
        if(i+1==j){
            return 0;
        }
        if(vis[i][j]!=0){
            return vis[i][j];//记忆化搜索
        }
        int ans=Integer.MAX_VALUE;
        //枚举k
        for(int k=i+1;k<=j-1;k++){
            ans=Math.min(ans,dfs(i,k,v)+dfs(k,j,v)+v[i]*v[j]*v[k]);
        }
        return vis[i][j]=ans;

    }
}

# 翻译成递推做法

首先将dfs()换成数组f

然后看循环顺序

  • ，从转移而来，所以是倒序枚举
    • 比较和
  • ,从转移而来，说明是顺序枚举

边界

  • 对于，由于两个顶点是，答案为零，所以 应该从开始枚举
    • 是一个顶点，是两个，才是至少3个顶点
  • 对于，由于, 从枚举到，所以最小情况下，并且

答案是

class Solution {
    public int minScoreTriangulation(int[] v) {
        int n=v.length;
        int f[][] = new int[n+1][n+1];
        for(int i=n-3;i>=0;i--){
            for(int j=i+2;j<n;j++){
                int ans=Integer.MAX_VALUE;
                for(int k=i+1;k<j;k++){
                    ans = Math.min(ans,f[i][k]+f[k][j]+v[i]*v[j]*v[k]);
                }
                f[i][j]=ans;
            }
        }
        return f[0][n-1];
    }
}

# 375. 猜数字大小 II

375. 猜数字大小 II (opens new window)

思路：

10分钟没思路，看题解

https://leetcode.cn/problems/guess-number-higher-or-lower-ii/solutions/1097127/gong-shui-san-xie-yi-ti-shuang-jie-ji-yi-92e5

  • 确保获胜，意味着每次都猜错。

    • 如果正确数字是，那么确保获胜的金额就是每次都猜错，直到区间缩小到​时的花费。

    对于有回应的猜数字，我们都知道是二分，​时间里可以猜到一个数。但是这里不是二分。

    原因：我们二分查找一个数，每次都可以缩小一半的范围，但是这里猜错了，不一定可以缩小一半的范围（也就是猜的不一定是mid），所以不应该是二分，而是要暴力所有情况找最大值。

  • 惩罚大小是，由于我们要确保胜利，那么我们这一次应当考虑花费最高的情况，确保胜利，这里应该是max

    • 在确保胜利的前提下，我们希望最终是最小的获胜现金数，所以应该最终是取min

启发思路：

假设真实的数是,其可能区间是，那么当前操作是可以从中任意选一个，选了这个之后，真实值可能在其左边，也可能在右边。为了确保胜利，这里我们取下一个可能区间中的最大值。

  • 所以可以转移到​

这里最需要关注的是从转移到了这更小的区间

由于希望最后是最小花费，所以我们枚举中的每个数当做，然后选其中最小的情况

这样做的确子问题和父问题性质一样，并且规模更小

状态定义：

定义表示在区间中任意选择一个数，确保能猜出来的最小花费

转移：

​

边界:

​​时，可以唯一确定真实值了，返回0

，区间为空，也应该返回0

class Solution {
    int [][]cache;
    public int getMoneyAmount(int n) {
        cache=new int[n+1][n+1];
        for(int i=0;i<n;i++) Arrays.fill(cache[i],-1);
        return dfs(1,n);
    }
    public int dfs(int i, int j){
        if(i>=j) return 0;
        if(cache[i][j]!=-1) return cache[i][j];
        int ans=Integer.MAX_VALUE;
        for(int x=i;x<=j;x++){
            int cur = Math.max(dfs(i,x-1),dfs(x+1,j))+x;
            ans = Math.min(ans,cur);
        }
        return cache[i][j]=ans;
    }
}

class Solution {
    public int getMoneyAmount(int n) {
        int [][]f=new int[n+2][n+2];
        for(int len=2;len<=n;len++){
            for(int l=1;l+len-1<=n;l++){
                int r=l+len-1;

                }
            }
        }
        return f[1][n];
    }
    
}

class Solution {
    public int minCut(String s) {
        int n = s.length();
        boolean[][] g = new boolean[n][n];
        for (int i = 0; i < n; ++i) {
            Arrays.fill(g[i], true);
        }

        for (int i = n - 1; i >= 0; --i) {
            for (int j = i + 1; j < n; ++j) {
                g[i][j] = s.charAt(i) == s.charAt(j) && g[i + 1][j - 1];
            }
        }

        int[] f = new int[n];
        Arrays.fill(f, Integer.MAX_VALUE);
        for (int i = 0; i < n; ++i) {
            if (g[0][i]) {
                f[i] = 0;
            } else {
                for (int j = 0; j < i; ++j) {
                    if (g[j + 1][i]) {
                        f[i] = Math.min(f[i], f[j] + 1);
                    }
                }
            }
        }

        return f[n - 1];
    }
}

  • 2，由于有可能导致负数下标，所以要替换为
class Solution {
    public int maxOperations(int[] nums) {
        int n=nums.length;
        int x=nums[0]+nums[n-1];
        int ans=0;
        ans=Math.max(ans,dp(nums,x,1,n-2));
        int y=nums[0]+nums[1];
        ans=Math.max(ans,dp(nums,y,2,n-1));
        int z=nums[n-2]+nums[n-1];
        ans=Math.max(ans,dp(nums,z,0,n-3));
        return ans+1;
        
    }
    public int dp(int []nums,int x,int l,int r){//左闭右闭区间
        int n=nums.length;
        int [][]f=new int[n+4][n+4];
        int res=0;
        for(int i=r;i>=l;i--){
            for(int j=i+1;j<=r;j++){
                if(x==nums[i]+nums[j]){
                    f[i][j+2]=Math.max(f[i][j+2],f[i+1][j+1]+1);
                }
                if(x==nums[i]+nums[i+1]){
                    f[i][j+2]=Math.max(f[i][j+2],f[i+2][j+2]+1);
                }
                if(x==nums[j-1]+nums[j]){
                    f[i][j+2]=Math.max(f[i][j+2],f[i][j]+1);
                }
            }
        }
        return f[l][r+2];
    }
}

# 1312. 让字符串成为回文串的最少插入次数

1312. 让字符串成为回文串的最少插入次数 (opens new window)

思路：

一般可以先反向思考一下会不会更简单

正难则反。既然我们要有最少操作次数的回文字符串，那么我们就应该保留尽量多的字符，这样操作次数最少。

所以应该找最长的回文子序列，然后用长度减去最长回文子序列长度即可？

Question:为什么这两个就是互补问题？最长回文子序列之外的字符就一定能随便转换为回文的吗？

Ans：我们假设有最长回文序列，然后原本的字符串中还有一个序列，无论其长度和位置都可以是最长回文子序列。

根据数学归纳法，那么推广到任意情况也正确。

class Solution {
    public int minInsertions(String s) {
        int n=s.length();
        int [][] f=new int[n][n];
        for(int i=0;i<n;i++) f[i][i]=1;
        for(int i=n-1;i>=0;i--){
            for(int j=i+1;j<n;j++){
                if(s.charAt(i)==s.charAt(j)){
                    f[i][j]=Math.max(f[i+1][j-1]+2,f[i][j]);
                }else{
                    f[i][j]=Math.max(f[i+1][j],f[i][j-1]);
                }
            }
        }
        return n-f[0][n-1];
    }
}

class Solution {
    int n,m;
    int [][]cache;
    int []nums;
    int []mult;
    public int maximumScore(int[] nums, int[] mult) {
        n=nums.length;
        m=mult.length;
        this.nums=nums;
        this.mult=mult;
        cache = new int[m][m];
        for(int i=0;i<m;i++){
            Arrays.fill(cache[i],Integer.MIN_VALUE);
        }
        return dfs(0,n-1,0);
    }
    public int dfs(int i,int j,int k){
        if(k==m||i>j) return 0;
        if(cache[i][k]!=Integer.MIN_VALUE) return cache[i][k];
        return cache[i][k]= Math.max(dfs(i+1,j,k+1)+nums[i]*mult[k],dfs(i,j-1,k+1)+nums[j]*mult[k]);
    }
}

# 1771. 由子序列构造的最长回文串的长度

1771. 由子序列构造的最长回文串的长度 (opens new window)

思路：

不会转换。直接看题解了

灵神题解：https://leetcode.cn/problems/maximize-palindrome-length-from-subsequences/solutions/2285215/shi-pin-qiao-miao-zhuan-huan-516-bian-xi-jhrt

既然是要分别从两个序列中选出最长子序列，那么好像可以记​，然后再去求s的最长回文序列。

那么我们有以下想法：

  • s的最长公共子序列就是所求序列吗？这是错误的，因为这里有一个限制条件是子序列不能为空，这样做忽略了这个条件

  • 在516:最长回文子序列中，我们表示从到的所有最长回文序列。那么我们只考虑和时的​情况，然后取其中最大值，这样可以保证我们从两个序列中都选了字符，但是还是错的

    • 因为这个起点就是错的，假设有序列word1="abcd",word="eee"。那么s="abcdeee"。那么满足我们的情况，但是他仍然是word1选了空，word2选择的是"eee"。

  • 正确做法是再上面的条件下，只有当时，才更新答案找最大值。因为这个条件下，才能保证包含包含了和。

class Solution {
    public int longestPalindrome(String word1, String word2) {
        char []s = (word1+word2).toCharArray();
        int ans=0,n=s.length;
        int n1=word1.length();
        int [][]f = new int[n][n];
        for(int i=n-1;i>=0;i--){
            f[i][i]=1;
            for(int j=i+1;j<n;j++){
                if(s[i]==s[j]){
                    f[i][j]=f[i+1][j-1]+2;
                    if(i<n1&&j>=n1){//注意边界<n1即<=n1-1
                    //>=n1才是n2的第一个下标
                        ans=Math.max(ans,f[i][j]);
                    }
                }else{
                    f[i][j]=Math.max(f[i+1][j],f[i][j-1]);
                }
            }
        }
        return ans;
    }
}

# 1547. 切棍子的最小成本

1547. 切棍子的最小成本 (opens new window)

思路：

启发思路：

对于长为的木棍，如果我们选择一个切点，那么这个木棍就会变成和。这样就已经变成了更小的子问题。

状态定义：

我们假设表示长的木棍的最小长度和。

转移方程：

如果其在点切分，那么

这里有个限制条件，由于有多个切点，所以我们枚举切点时，应该满足，这也是后面优化的依据。

边界：

如果​​，已经切不了了，返回0

转移方向

这里注意表明上好像没有转移，但是实际上是左边，​​是右边。

这里最好想的是像[3040]这道题一样，第一维枚举长度，第二维枚举左端点，然后右端点可以推算出来，

等价于逆序，正序，也是一样得效果。

超时的答案，也会爆内存

class Solution {
    public int minCost(int n, int[] cuts) {
        int[][] dp = new int[n + 1][n + 1];//n=10^6

        for(int i = n - 1; i >= 0; i--) {
            for(int j = i + 1; j < n + 1; j++) {
                int ans = Integer.MAX_VALUE / 10;
                for(int k : cuts) {
                    // 如果切点在(i, j)之内, 尝试切分
                    if(i < k && k < j) {
                        ans = Math.min(ans, dp[i][k] + dp[k][j] + j - i);
                    }
                }
                // 如果没有切分则没有成本, 所以不能取ans
                dp[i][j] = ans == Integer.MAX_VALUE / 10 ? 0 : ans;
            }
        }
        return dp[0][n];
    }
}

class Solution {
    public int minCost(int n, int[] cuts) {
        int m=cuts.length;
        int []list = new int[m+2];
        list[0]=0;
        list[m+1]=n;
        Arrays.sort(cuts);
        int[][] dp = new int[m+2][m+2];
        for(int i=0;i<m;i++){
            list[i+1]=cuts[i];
        }
        for(int i = m+1; i >= 0; i--) {
            for(int j = i + 1; j <=m+1; j++) {
                int ans = Integer.MAX_VALUE / 2;
                for(int k=i+1;k<j;k++) {
                    //这里去掉了判断，因为切点一定在里面
                        ans = Math.min(ans, dp[i][k] + dp[k][j] + list[j]- list[i]);
             
                }
                // 如果没有切分则没有成本, 所以不能取ans
                dp[i][j] = ans == Integer.MAX_VALUE / 2? 0 : ans;
            }
        }
        return dp[0][m+1];
    }
}
