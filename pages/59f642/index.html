分组循环

灵神题单 (opens new window)

# 定义

# 适用场景

如果一个数组，需要分成若干组，并且每一组的判断/处理逻辑是相同的（也就是说每一组都有功能的性质，最常见的就是相等/升序/降序）

一般分为外层循环和内层循环，两个循环用于处理不同的事情

1.外层循环用于做准备工作（比如记录开始位置）和内层循环后的统计工作（例如求最大值）

2.内层循环用于遍历组，找出这个组的结束位置

分组循环这种写法逻辑清晰，不容易出错

# 例题

2765. 最长交替子数组 (opens new window)

思路：

  • 外层循环负责遍历组之前的准备工作（记录开始位置），和遍历组之后的统计工作（更新答案最大值）。
  • 内层循环负责遍历组，找出这一组最远在哪结束。

注意：

1.对于交替两个字，可以转化成nums[i]=nums[i+2]

2.交替数组的起点是1

3.不存在解返回-1，所以ans初始化为-1；

class Solution {
    public int alternatingSubarray(int[] nums) {
        int ans =-1;
        int i=0,n=nums.length;
        while(i<n-1){
            if(nums[i+1]-nums[i]!=1){//交替数组第一组必须是1
                i++;
                continue;
            }
            int j=i;//记录这一组开始位置
            i+=2;//i和i+1已经满足
            while(i<n&&nums[i]==nums[i-2]){//交替可以转换为这个条件
                i++;
            }
            //到这里，i指向的数是第一个不是交替数组的数
            //从j到i-1是满足要求的
            ans=Math.max(ans,i-j);
            i--;//这里很细节，注意，因为i所处的点是第一个不是交替数组的数，所以要更新为交替数组的最后一个数
            //可以看灵神题解
        }
        return ans;
    }
}

# 1446. 连续字符

1446. 连续字符 (opens new window)

这道题中，按照种类给字符串分组，求出最长的组。

思路： 外层循环： 记录开始位置，组长度 内层循环 遍历该组，找结束位置跳出 注意： 1.如果要比较i和i+1个元素，那么i应该小于n-1;

class Solution {
    public int maxPower(String s) {
       int ans =1;
       int i=0,n=s.length();
       while(i<n){
           int j=i;
           while(j<n-1&&s.charAt(j)==s.charAt(j+1)){
               j++;
           }
        ans = Math.max(ans,j-i+1);
           i=j+1;
       }
        return ans;
    }
}

# 1869. 哪种连续子字符串更长

1869. 哪种连续子字符串更长 (opens new window)

思路： 外层循环： 分别记录1开始位置，和0开始长度，并且更新其最长长度 内层循环 遍历该组，找结束位置跳出（也就是和上一个不相等）

class Solution {
    public boolean checkZeroOnes(String s) {
        int n=s.length();
        if(n==1){
            return s.charAt(0)=='1';
        }
        int one_len =getLen(s,'1',n);
        int zero_len = getLen(s,'0',n);
        return one_len>zero_len;
    }
    public int getLen(String s,char ch,int n){
        int i=0;
        int len=0;
        while(i<n){
            int j=i;
            //这里比较的是连续，所以样例长度为1会有bug
            while(j<n-1&&s.charAt(j)==ch&&s.charAt(j+1)==ch){
                j++;
            }
            //假设ch=0，j停留在连续0的最后一位。
            len = Math.max(len,j-i+1);
            i=j+1;
        }
        return len;
    }
}

# 1957. 删除字符使字符串变好

1957. 删除字符使字符串变好 (opens new window)

贪心+分组循环

只要把长度为3以上的字符串替换成长度为2的就可以了。

思路：

外层循环 标记起点，更新sb 内层循环 标记连续数组终点

class Solution {
    public String makeFancyString(String s) {
        StringBuilder sb = new StringBuilder();
        int n=s.length();
        int i=0;
        while(i<n){
            int j=i;
            while(j<n-1&&s.charAt(j+1)==s.charAt(j)){
                j++;
            }
            //j停在连续字符串最后一位
            int len = j-i+1;
            if(len<3){
                sb.append(s.substring(i,j+1));
            }else{
                sb.append(  s.substring(i,i+2));//截取两个。
            }
            i=j+1;
        }
        return sb.toString();
    }
}

# 2110. 股票平滑下跌阶段的数目

2110. 股票平滑下跌阶段的数目 (opens new window)

首先每一天都算是一次平滑下降。 其次是对于[3,2,1]的这种长度超过3的下降序列，平滑下降阶段规律如下： 4 3 2 1 43 32 21 432 321 4321 总共有4+ 1+2+3= 10

54321= 5+ 1 +2+3+4=15。 54 43 32 21 543 432 321 5432 4321 54321 假设下降序列长度是n(n>2)，除开固有的n个阶段，其提供下降数为 1+2+3...+n-1 。根据等差求和公式为(n-1)(1+n-1)/2=(n)(n-1)/2 即

难得完全靠自己做出来一道数学题，也没啥bug，感谢灵神！

class Solution {
    public long getDescentPeriods(int[] prices) {
        int n=prices.length;
        long ans=0;
        ans+=n;
        long []pre =new long [n+1];
        for(int i=0;i<n;i++){//pre[i]表示0-i-1的和
            pre[i+1]=pre[i]+i;
        }
        int i=0;
        while(i<n){
            int j=i;
            while(j<n-1&&prices[j+1]==prices[j]-1){
                j++;   
            }
            //出来是j是具有这个性质序列最后一位
            int len=j-i+1;
            ans+=pre[len];
            i=j+1;
        }
        return ans;
        
    }
    
}

# 228.汇总区间

228. 汇总区间 (opens new window)

数组可分组，每组性质为刚好按1递增 外层循环 记录开始位置，输出区间范围 内层循环 寻找该组结束位置

class Solution {
    public List<String> summaryRanges(int[] nums) {
        int n=nums.length;
        int i=0;
        List<String>ans = new ArrayList<>();
        while(i<n){
            int j=i;
            while(j<n-1&&nums[j+1]==nums[j]+1){
                j++;
            }
            StringBuilder sb = new StringBuilder();
            sb.append(i==j?nums[i]:nums[i]+"->"+nums[j]);
            ans.add(sb.toString());
            i=j+1;
        }
        return ans;
    }
}

# 2760. 最长奇偶子数组

2760. 最长奇偶子数组 (opens new window)

数组可分组，给定一个threshold，每组性质是是每个元素i 奇偶交替，并且都小于threshold。 注意: 1.奇偶交替可以转换为连续两个数的和%2不为0 2.第一个数必须小于threshold,并且以偶数开头

外层循环： 记录开始位置，更新最长长度 内层循环： 寻找结束位置，根据每组性质

class Solution {
    public int longestAlternatingSubarray(int[] nums, int threshold) {
        int i=0;
        int n=nums.length;
        int ans=0;
    
        while(i<n){
            int j=i;
            if(nums[j]<=threshold&&nums[j]%2==0){
                while(j<n-1&&(nums[j]+nums[j+1])%2!=0 &&nums[j+1]<=threshold ){
                    j++;
                }
                //这种方法有个问题，就是ans至少会为1导致本来该是0的误判
                //比如[1]这种情况
                //因此把更新放到if里面
                ans = Math.max(j-i+1,ans);
            }
            i=j+1;
        }
        return ans;
    }
}

# 1887. 使数组元素相等的减少操作次数

1887. 使数组元素相等的减少操作次数 (opens new window)

由于只能减少，所以最后一定是所有元素都变成最小元素。 假设数组是[5,5,1,3],根据题目规则，要想每个元素都到最小元素， 那有 5 - 3 - 1 5 - 3 - 1 3 -1,需要5步。 我们发现，元素x离其更小元素y的步数，就是看降序排列之后，x到y之间不相等的元素种类， 比如[5,5,3,1], 我们从5到3， 有 5 ， 3 ， 1；所以要两步。 从3 到1 有 3, 1 ，所以只需要一步

结论： 我们先找到最小元素，然后统计所有比最小元素大的元素到最小元素的步数即可。 并且元素相等时，下标先后对于操作步数没有影响，可以忽略。 具体实现：排序后分组循环 为了方便，我们可以升序排列数组，然后进行分组循环，维护一个步数x，表示从第y组元素到第一个元素所需步数，每次加上更新后的x就是ans。

每组性质是元素一样。 外层循环： 标记每组起点，维护步数y，更新答案ans 内层循环： 寻找每组终点。

class Solution {
    public int reductionOperations(int[] nums) {

        int n=nums.length;
        Arrays.sort(nums);//升序排列
        int step=0;
        int i=0;
        int ans=0;
        while(i<n){
            int j=i;
            while(j<n-1&&nums[j+1]==nums[j]){
                j++;
            }
            ans+=step*(j-i+1);
            step++;//这里很细节，因为最小元素到最小元素显然是0步，所以是先ans更新，才step++，非常优雅
            i=j+1;
        }
        return ans;
    }
}

# 2038. 如果相邻两个颜色均相同则删除当前颜色

2038. 如果相邻两个颜色均相同则删除当前颜色 (opens new window)

灵神题解 (opens new window)

这道题是脑筋急转弯，理解题意挺有难度的

主要是读题比较困难。 题意指的是，Alice每次能删1个A，Bob每次能删1个B， 并且对于这个片段，也不能删除两端，只能删除中间的A/B。

如果只有AA/BB,由于没有中间的字符，所以操作不了。

这里很重要，因为不能删除两端的AA/BB，所以分散的A和B不会连起来。 这里必须从这些推导出一个重要结论 因此删除一种颜色不会对另一种颜色产生任何影响，我们只需要统计每一串连续相同颜色的长度 l，如果l>2,则可以删除l-2个颜色。

class Solution {
    public boolean winnerOfGame(String colors) {
        int n= colors.length();
        int i=0,x=0,y=0;
        while(i<n){
            int j=i;
            while(j<n-1&&colors.charAt(j)==colors.charAt(j+1)){
                j++;
            }
            if(j-i+1>2){
                if(colors.charAt(i)=='A'){
                    x+=j-i-1;//长度减2
                }else{
                    y+=j-i-1;
                }
            }
            i=j+1;
        }
        return x>y;
    }
}
